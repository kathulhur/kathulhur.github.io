About

I build software the way I'd want to use it — clear, fast, and honest about its edges.

My work sits at the intersection of engineering and design. I like problems that need both a solid data model and a considered interface, and I care as much about the quiet details — empty states, loading, error copy — as the headline features.

When I'm not building, I'm usually reading about systems design, refining my own tools, or exploring on foot.
